# Repo Standards Publication Plan

## Purpose

Define how Publisher packages display-only guidance for StegVerse-Labs repo-standards.

## Source

- Repository: StegVerse-Labs/repo-standards
- State: 0.1.0-rc.1 release-candidate surface prepared
- Site mirror: StegVerse-Labs/Site repo-standards.html

## Publication Boundary

Publisher does not own or validate repository standards.

Publisher may package a summary, adoption path, and downstream publication status.

Publisher must keep the source boundary visible:

- repo-standards is authoritative for standards files
- Site is display-only
- Publisher is packaging and distribution support
- wiki targets remain pending until separately installed

## Publication Contents

A publication package should include:

1. source repository name
2. source version or candidate state
3. standards list ST-001 through ST-013
4. adoption path
5. validation boundary
6. downstream pending targets

- stegguardian-wiki

Publisher is ready to package repo-standards guidance as display-only release-candidate information. It is not authorized to mark the standards as validated release.
