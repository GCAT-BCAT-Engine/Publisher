## Behavior

The scheduled Publisher workflow acquires the Site propagation packet, verifies that Publisher is a declared destination, checks the packet's canonical hash when supplied, rejects any authority escalation, and persists a repository-owned awareness receipt.

Before Site activation completes, Publisher records:

```text
state: PENDING_SITE_ACTIVATION
blockers: non-empty
```

Only when Site publishes:

```text
state: READY_FOR_DOWNSTREAM_INGESTION
```

may Publisher record:

```text
state: VERIFIED_INGESTION_READY
```

Neither state grants publication, release, activation, admissibility, or execution authority.

## Machine-owned continuation

1. Run the Publisher awareness workflow hourly.
2. Persist the current Site propagation state without manual dispatch.
3. Retain a non-empty blocker list while Site is pending.
4. Reject destination drift, hash mismatch, or authority escalation.
5. When Site becomes ready, persist `VERIFIED_INGESTION_READY` without converting awareness into publication authority.
6. Update the primary Publisher mirror handoff after the first observed workflow receipt.

## Release posture

No tag or release is authorized by this consumer. It installs downstream readiness awareness only.
